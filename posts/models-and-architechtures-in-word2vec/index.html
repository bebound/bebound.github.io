Models and Architectures in Word2vec

Generally, word2vec is a language model to predict the words probability based on the context. When build the model, it create word embedding for each word, and word embedding is widely used in many NLP tasks.

Models

CBOW (Continuous Bag of Words)

Use the context to predict the probability of current word. (In the picture, the word is encoded with one-hot encoding, W_V\*NW\_{V\*N}W_V\*N is word embedding, and W_V\*N′W\_{V\*N}^{'}W_V\*N′, the output weight matrix in hidden layer, is same as υ^\hat{\upsilon}υ^ in following equations)

  1. Context words’ vectors are υ_c−n...υ_c+m\upsilon\_{c-n} ... \upsilon\_{c+m}υ_c−n...υ_c+m (mmm is the window size)
  2. Context vector υ^=υ_c−m+υ_c−m+1+...+υ_c+m2m\hat{\upsilon}=\frac{\upsilon\_{c-m}+\upsilon\_{c-m+1}+...+\upsilon\_{c+m}}{2m}υ^=2mυ_c−m+υ_c−m+1+...+υ_c+m​
  3. Score vector z_i=u_iυ^z\_i = u\_i\hat{\upsilon}z_i=u_iυ^, where u_iu\_iu_i is the output vector representation of word ω_i\omega\_iω_i
  4. Turn scores into probabilities y^=softmax(z)\hat{y}=softmax(z)y^​=softmax(z)
  5. We desire probabilities y^\hat{y}y^​ match the true probabilities yyy.

We use cross entropy H(y^,y)H(\hat{y},y)H(y^​,y) to measure the distance between these two distributions.

H(y^,y)=−∑_j=1∣V∣y_jlog⁡(y^_j)H(\hat{y},y)=-\sum\_{j=1}^{\lvert V \rvert}{y\_j\log(\hat{y}\_j)}H(y^​,y)=−∑_j=1∣V∣y_jlog(y^​_j)

yyy and y^\hat{y}y^​ is accurate, so the loss simplifies to:

H(y^,y)=−y_jlog⁡(y^)H(\hat{y},y)=-y\_j\log(\hat{y})H(y^​,y)=−y_jlog(y^​)

For perfect prediction, H(y^,y)=−1log⁡(1)=0H(\hat{y},y)=-1\log(1)=0H(y^​,y)=−1log(1)=0

According to this, we can create this loss function:

  1. We get the input word’s vector υ_c\upsilon\_cυ_c

  4. We desire probabilities y^\hat{y}y^​ match the true probabilities yyy.

Architectures

Minimize JJJ is expensive, you need to calculate the probability of each word in vocabulary list. There are two ways to reduce the computation. Hierarchical Softmax and Negative Sampling.

Hierarchical Softmax

Encode words into a huffman tree, then each word has a Huffman code. The probability of it’s probability P(w∣Context(ω))P(w\lvert Context(\omega))P(w∣Context(ω)) can change to choose the path from root to the leaf node, each node is a binary classification. Suppose code 000 is a positive label, 111 is negative label. If the probability of a positive classification is

σ(XT_ωθ)=11+e−XT_ω\sigma(X^T\_\omega \theta)=\frac{1}{1+e^{-X^T\_\omega}}σ(XT_ωθ)=1+e−XT_ω1​

Then the probability of negative classification is

1−σ(XT_ωθ)1-\sigma(X^T\_\omega \theta)1−σ(XT_ωθ)

足球’s Huffman code is 100110011001, then it’s probability in each node are

p(d_2ω∣X_ω,θω_1=1−σ(XT_ωθω_1))p(dω_3∣X_ω,θω_2=σ(XT_ωθω_2))p(dω_4∣X_ω,θω_3=σ(XT_ωθω_3))p(dω_5∣X_ω,θω_4=1−σ(XT_ωθω_4))\begin{aligned} p(d\_2^\omega\lvert X\_\omega,\theta^\omega\_1&=1-\sigma(X^T\_\omega \theta^\omega\_1))\\\\ p(d^\omega\_3\lvert X\_\omega,\theta^\omega\_2&=\sigma(X^T\_\omega \theta^\omega\_2))\\\\ p(d^\omega\_4\lvert X\_\omega,\theta^\omega\_3&=\sigma(X^T\_\omega \theta^\omega\_3))\\\\ p(d^\omega\_5\lvert X\_\omega,\theta^\omega\_4&=1-\sigma(X^T\_\omega \theta^\omega\_4))\\\\ \end{aligned}p(d_2ω∣X_ω,θω_1p(dω_3∣X_ω,θω_2p(dω_4∣X_ω,θω_3p(dω_5∣X_ω,θω_4​=1−σ(XT_ωθω_1))=σ(XT_ωθω_2))=σ(XT_ωθω_3))=1−σ(XT_ωθω_4))​

Negative Sampling

This method will choose some negative sample, then add the probability of the negative word into loss function. The optimisation target becomes maximise the positive words’ probability and minimise the negative words’ probability.

Let P(D=0∣ω,c)P(D=0 \lvert \omega,c)P(D=0∣ω,c) be the probability that (ω,c)(\omega,c)(ω,c) did not come from the corpus data. Then the objective function will be

θ=argmax∏_(ω,c)∈DP(D=1∣ω,c,θ)∏_(ω,c)∈D~P(D=0∣ω,c,θ)\theta = \text{argmax} \prod\_{(\omega,c)\in D} P(D=1\lvert \omega,c,\theta) \prod\_{(\omega,c)\in \tilde{D}} P(D=0\lvert \omega,c,\theta)θ=argmax∏_(ω,c)∈DP(D=1∣ω,c,θ)∏_(ω,c)∈D~P(D=0∣ω,c,θ)

where θ\thetaθ is the parameters of the model(υ\upsilonυ and uuu).

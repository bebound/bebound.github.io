Generally, word2vec is a language model to predict the words probability based on the context. When build the model, it create word embedding for each word, and word embedding is widely used in many NLP tasks.

Models

CBOW (Continuous Bag of Words)

Use the context to predict the probability of current word. (In the picture, the word is encoded with one-hot encoding, WV∗NW_{V*N}WV∗N​ is word embedding, and WV∗N′W_{V*N}^{'}WV∗N′​, the output weight matrix in hidden layer, is same as υ^\hat{\upsilon}υ^ in following equations)

  1. Context words’ vectors are υc−n...υc+m\upsilon_{c-n} ... \upsilon_{c+m}υc−n​...υc+m​ (mmm is the window size)
  2. Context vector υ^=υc−m+υc−m+1+...+υc+m2m\hat{\upsilon}=\frac{\upsilon_{c-m}+\upsilon_{c-m+1}+...+\upsilon_{c+m}}{2m}υ^=2mυc−m​+υc−m+1​+...+υc+m​​
  3. Score vector zi=uiυ^z_i = u_i\hat{\upsilon}zi​=ui​υ^, where uiu_iui​ is the output vector representation of word ωi\omega_iωi​
  4. Turn scores into probabilities y^=softmax(z)\hat{y}=softmax(z)y^​=softmax(z)
  5. We desire probabilities y^\hat{y}y^​ match the true probabilities yyy.

We use cross entropy H(y^,y)H(\hat{y},y)H(y^​,y) to measure the distance between these two distributions.

H(y^,y)=−∑j=1∣V∣yjlog⁡(y^j)H(\hat{y},y)=-\sum_{j=1}^{\lvert V \rvert}{y_j\log(\hat{y}_j)}H(y^​,y)=−j=1∑∣V∣​yj​log(y^​j​)

H(y^,y)=−yjlog⁡(y^)H(\hat{y},y)=-y_j\log(\hat{y})H(y^​,y)=−yj​log(y^​)

Architectures

Minimize JJJ is expensive, you need to calculate the probability of each word in vocabulary list. There are two ways to reduce the computation. Hierarchical Softmax and Negative Sampling.

Hierarchical Softmax

Encode words into a huffman tree, then each word has a Huffman code. The probability of it’s probability P(w∣Context(ω))P(w\lvert Context(\omega))P(w∣Context(ω)) can change to choose the path from root to the leaf node, each node is a binary classification. Suppose code 000 is a positive label, 111 is negative label. If the probability of a positive classification is

σ(XωTθ)=11+e−XωT\sigma(X^T_\omega \theta)=\frac{1}{1+e^{-X^T_\omega}}σ(XωT​θ)=1+e−XωT​1​

Then the probability of negative classification is

1−σ(XωTθ)1-\sigma(X^T_\omega \theta)1−σ(XωT​θ)

足球’s Huffman code is 100110011001, then it’s probability in each node are

p(d2ω∣Xω,θ1ω=1−σ(XωTθ1ω))p(d3ω∣Xω,θ2ω=σ(XωTθ2ω))p(d4ω∣Xω,θ3ω=σ(XωTθ3ω))p(d5ω∣Xω,θ4ω=1−σ(XωTθ4ω))\begin{aligned} p(d_2^\omega\lvert X_\omega,\theta^\omega_1&=1-\sigma(X^T_\omega \theta^\omega_1))\\ p(d^\omega_3\lvert X_\omega,\theta^\omega_2&=\sigma(X^T_\omega \theta^\omega_2))\\ p(d^\omega_4\lvert X_\omega,\theta^\omega_3&=\sigma(X^T_\omega \theta^\omega_3))\\ p(d^\omega_5\lvert X_\omega,\theta^\omega_4&=1-\sigma(X^T_\omega \theta^\omega_4))\\ \end{aligned}p(d2ω​∣Xω​,θ1ω​p(d3ω​∣Xω​,θ2ω​p(d4ω​∣Xω​,θ3ω​p(d5ω​∣Xω​,θ4ω​​=1−σ(XωT​θ1ω​))=σ(XωT​θ2ω​))=σ(XωT​θ3ω​))=1−σ(XωT​θ4ω​))​

Negative Sampling

This method will choose some negative sample, then add the probability of the negative word into loss function. The optimisation target becomes maximise the positive words’ probability and minimise the negative words’ probability.

Let P(D=0∣ω,c)P(D=0 \lvert \omega,c)P(D=0∣ω,c) be the probability that (ω,c)(\omega,c)(ω,c) did not come from the corpus data. Then the objective function will be

θ=argmax∏(ω,c)∈DP(D=1∣ω,c,θ)∏(ω,c)∈D~P(D=0∣ω,c,θ)\theta = \text{argmax} \prod_{(\omega,c)\in D} P(D=1\lvert \omega,c,\theta) \prod_{(\omega,c)\in \tilde{D}} P(D=0\lvert \omega,c,\theta)θ=argmax(ω,c)∈D∏​P(D=1∣ω,c,θ)(ω,c)∈D~∏​P(D=0∣ω,c,θ)

where θ\thetaθ is the parameters of the model(υ\upsilonυ and uuu).
